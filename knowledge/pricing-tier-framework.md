# Pricing Tier Framework

Structure, anchoring strategy, and content rules for three-tier proposal pricing.

---

## Tier Definitions

| Tier | Name | Purpose | Target Win Rate |
|------|------|---------|-----------------|
| 1 | Essential | Minimum viable delivery; anchors low end | 15-20% |
| 2 | Professional | Full scope + recommended additions; the target tier | 55-65% |
| 3 | Enterprise | Full scope + premium options; anchors high end | 15-25% |

The Professional tier is always the intended selection. Essential exists to anchor the low end and make Professional look reasonable. Enterprise exists to anchor the high end and make Professional look like the smart choice.

---

## Tier Content Rules

### Essential

- Core deliverables only — the minimum that solves the stated problem
- No optional features, extended support, or premium SLAs
- Standard timeline (no acceleration)
- Basic training / knowledge transfer
- Limited revision rounds (1-2)
- Payment: milestone-based or 50/30/20

### Professional

- Full scope with all recommended features
- Includes items the client expects but did not explicitly request
- Accelerated timeline where feasible
- Comprehensive training and documentation
- Standard revision rounds (2-3)
- Extended warranty / support period (30-90 days post-delivery)
- Payment: milestone-based with retainer option

### Enterprise

- Everything in Professional
- Premium additions: extended support (6-12 months), SLA guarantees, dedicated account manager
- Training program (multi-session, multi-audience)
- Priority response times
- Custom integrations or advanced features
- Unlimited revision rounds within scope
- Payment: retainer or annual contract

---

## Pricing Psychology

### Anchoring Rules

1. Present tiers in order: Essential → Professional → Enterprise
2. Professional price should be 1.5-2x Essential
3. Enterprise price should be 2-3x Essential
4. Highlight Professional tier visually (bold, "Recommended" badge, or emphasis)
5. Feature comparison table with checkmarks shows Professional as the value sweet spot

### Price Presentation

| Pattern | Example |
|---------|---------|
| Range format | "$150,000 - $180,000" |
| Monthly equivalent | "$15,000/month over 12 months" |
| Per-unit context | "$1,200 per developer seat" |
| Comparison anchor | "vs. $X annual cost of current approach" |

---

## Tier Differentiation Checklist

Each tier must differ on at least 4 of these dimensions:

| Dimension | Essential | Professional | Enterprise |
|-----------|-----------|--------------|------------|
| Feature scope | Core only | Full | Full + premium |
| Support duration | 0-30 days | 30-90 days | 6-12 months |
| SLA | None | Standard | Premium |
| Training | Docs only | 1 session | Multi-session program |
| Revisions | 1-2 rounds | 2-3 rounds | Unlimited in scope |
| Timeline | Standard | Standard or accelerated | Accelerated |
| Dedicated PM | No | Shared | Dedicated |
| Custom work | No | Limited | Yes |

---

## What NOT to Do

- Never present a single price point — forces binary yes/no decision
- Never make Essential so weak it looks like a throwaway option
- Never make Enterprise so expensive it looks unreasonable
- Never hide what differs between tiers — explicit comparison table required
- Never price before scope is defined — pricing is a function of scope, not a guess
- Never present pricing without the ROI section appearing first in the document
